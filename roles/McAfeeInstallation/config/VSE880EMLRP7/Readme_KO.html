이 문서 정보

VirusScan Enterprise 소프트웨어를 사용해 주셔서 감사합니다. 이 문서에는 현재 릴리스에 대한 중요한 정보가 들어 있습니다. 문서 전체를 완전히 읽고 이해하는 것이 좋습니다.

주의: 이 소프트웨어의 릴리스 이전 버전은 자동 업그레이드가 지원되지 않습니다. 소프트웨어의 제품 릴리스 버전으로 업그레이드하려면 소프트웨어의 기존 버전을 먼저 제거해야 합니다.

VirusScan Enterprise 8.8 릴리스에는 다음 항목이 사용됩니다.

  • 엔진 버전: 5400.1158
  • 탐지 정의(DAT): 이 버전의 VirusScan Enterprise 소프트웨어는 소프트웨어가 설치될 때 최신 버전의 DAT 파일을 자동으로 다운로드합니다. 이전에 DAT 파일은 설치 실행 파일에 포함되어 있었습니다.

새로운 기능

VirusScan Enterprise 8.8 소프트웨어 릴리스의 새로운 기능과 업데이트된 기능

성능 향상

이 릴리스에서는 다음이 감소합니다.
  • 온 액세스 검색 시간
  • 주문형 검색 시간
  • 시스템 시작 시간

중복 파일 검색을 줄이고 성능을 향상시키기 위해 이전에 검색된 파일을 공통 위치에 캐시합니다.

Windows Office 2010 지원

이 릴리스에서는 특히 Microsoft Outlook 2010 전자 메일 및 첨부 파일의 온액세스 검색을 추가하여 모든 Windows Office 2010 응용프로그램을 지원합니다.

ScriptScan 제외 지원

알려진 안전한 웹 사이트에서 신뢰할 수 있는 스크립트의 URL인 ScriptScan 제외를 쉽게 추가하여 검색을 줄이고 성능을 향상시킬 수 있습니다. 이 기능은 ePolicy Orchestrator 4.0 이상 및 VirusScan 콘솔에서 사용할 수 있습니다. 이전에 이러한 제외된 URL에는 수동 레지스트리 편집이 필요했습니다.

ePolicy Orchestrator에 정책 및 이벤트 마이그레이션

  • 문제

    ePolicy Orchestrator 마이그레이션 도구는 ePolicy Orchestrator 액세스에 Windows NT 인증을 사용하는 ePolicy Orchestrator 서버에서 기존 정책과 작업을 마이그레이션하지 못할 수 있습니다. 경우에 따라 ePolicy Orchestrator 마이그레이션 도구는 SQL 관리자에게 ePolicy Orchestrator 마이그레이션 도구의 매개 변수로 암호를 전달해야 합니다.

    해결 방법: 명령줄에서 ePOPolicyMigration.exe 명령을 실행하고 /PASSWORD 스위치와 SQL 관리자 암호를 추가합니다. 예를 들어, 다음과 같이 입력합니다. [path]\epopolicymigration.exe /password=sql_admin_password

    명령줄에서 ePolicy Orchestrator /PASSWORD 스위치 사용에 대한 자세한 내용은 McAfee VirusScan Enterprise 8.8.0 설치 안내서를 참조하십시오. (611047)

지원되는 플랫폼 및 제품

  • 이 VirusScan Enterprise 버전에서는 Lotus Notes 버전 7.0.x 및 8.0.x – 8.5.1을 지원합니다. 지원되는 응용프로그램에 대한 자세한 내용은 VirusScan Enterprise 8.8 설치 안내서를 참조하십시오.

다른 제품과의 호환성

  • 문제
    ePolicy Orchestrator 사용자 인터페이스의 처음 수행할 액션 목록에서 액션에 대한 프롬프트 작업 선택이 없습니다. 예를 들어 ePolicy Orchestrator 사용자 인터페이스를 사용하여 주문형 검색 작업을 구성하려면 다음을 수행합니다.
    • 시스템 트리 | 클라이언트 작업 | VSE 8.8.0에 대한 주문형 검색 작업을 클릭합니다.
    • 설정 편집, 구성 및 액션 탭을 클릭합니다.

    이 대화 상자의 처음 수행할 액션 목록에 액션에 대한 프롬프트 작업 선택이 없습니다.

    VirusScan 콘솔 사용자 인터페이스에서 액션에 대한 프롬프트 작업 선택을 사용할 수 있습니다. 예를 들면 다음과 같습니다.
    • 주문형 검색 작업을 두 번 클릭하는 경우 주문형 검색 등록 정보 대화 상자에서 액션 탭을 클릭합니다.
    • 처음 수행할 액션 목록에서 액션에 대한 프롬프트 작업을 선택할 수 있습니다.

    해결 방법: 없음. ePolicy Orchestrator 관리 검색 작업이므로 주문형 검색 작업이 실행되는 동안 사용자에게 표시되는 사용자 인터페이스가 없습니다. 따라서 사용자에게 액션을 요청하는 프롬프트를 표시하도록 구성할 수 없습니다. (590803)

  • 문제

    VirusScan Enterprise 8.7에서 VirusScan Enterprise 8.8로 업그레이드한 후 ePolicy Orchestrator 4.5 시스템 세부 정보 대화 상자에서 VirusScan Enterprise 8.8이 설치된 제품 행에 표시되지 않습니다. (620509)

  • 문제
    McAfee 네트워크 접근 제어 3.0이 설치된 시스템에 VirusScan Enterprise 8.8을 설치한 후 McAfee 서비스의 식별을 차단하는 실패가 발생할 수 있습니다. 이러한 증상을 볼 수 있습니다.
    • 다시 시작하기 전에 업데이트 작업을 수행하는 경우 지정된 드라이버가 올바르지 않음 메시지가 기록됩니다.
    • 하나 이상의 제품에 대해 드라이버를 로드하지 못함 메시지가 기록됩니다.
    • 로컬 시스템 응용프로그램 이벤트 로그에 이벤트 ID 5004가 있습니다.

    해결 방법: 실패한 제품을 제거하고 시스템을 다시 시작한 다음 제품을 다시 설치합니다.

  • 문제

    NVIDIA® 드라이버가 성능 문제나 시스템 실패를 초래할 수 있습니다. VirusScan Enterprise 8.8이 100% CPU 사용률로 실행되거나 시스템 실패를 초래할 수 있습니다. 이 문제에 대한 자세한 내용은 기술 자료 문서 KB614212 및 KB65066을 참조하십시오. (388060)

  • 문제

    VirusScan Enterprise 8.8 설치 프로그램을 실행할 경우 모든 CA Anti-Virus 버전 7.0 및 7.1 프로그램 디렉터리 구조와 레지스트리 키 항목을 완전히 제거하지 못합니다.

    해결 방법: 없음. CA Anti-Virus 제거 프로그램의 문제입니다. (625703)

  • 문제

    독립 실행형 클라이언트 시스템에서 VirusScan Enterprise 8.8이 함께 설치된 Panda Anti-Virus 2008의 초기 제품 업데이트를 완료하지 못합니다.

    VSESetup.exe를 실행할 때 Panda Anti-Virus 2008 소프트웨어를 제거하지 않도록 아니오 옵션을 선택한 경우 VirusScan Enterprise 8.8이 제품 업데이트를 수행하는 최종 단계에서 업데이트 대화 상자가 20분 이상 정지할 수 있습니다.

    해결 방법: VSESetup.exe를 실행할 때 예 옵션을 선택하여 Panda Anti-Virus 2008 소프트웨어를 제거합니다. (626009)

  • 문제

    ePolicy Orchestrator 관리 클라이언트에서 VSESetup.exe를 실행하고 탐지된 안티바이러스 소프트웨어를 제거하는 옵션을 선택하더라도 일부 Symantec 안티바이러스 파일, 디렉토리 및 레지스트리 키가 시스템에 남습니다.

    해결 방법: 없음. (629265)

업데이트

  • 문제
    업데이트 로그에서 “업데이트 실패”를 보고하고 다른 오류를 표시하지 않을 수 있습니다. 패치, DAT 및 엔진의 업데이트 기능을 사용하려면 업그레이드 패키지를 확장할 충분한 사용 가능한 공간과 메모리가 있어야 합니다. 업데이트에 성공하려면 다음의 최소 공간 요구 사항이 있어야 합니다.
    • 125MB의 사용 가능한 하드 디스크 공간
    • 250MB의 총 사용 가능한 하드 디스크 및 메모리 공간
      참고: Windows 가상 메모리를 사용하여 추가 메모리 공간을 생성할 수도 있지만 이 합계는 250MB의 사용 가능한 공간과 같아야 합니다.

    이 하드 디스크 및 메모리 요구 사항은 특히 최소 시스템 요구 사항을 실행하는 포함된 시스템과 가상 시스템에 중요합니다. (625749)

  • 문제

    리포지토리 액세스 실패는 VirusScan Enterprise 8.8 업데이트 로그에 기록되지 않지만 McAfee 에이전트 로그에 기록됩니다. McAfee 에이전트 로그의 기본 위치는 다음과 같습니다. <drive>:\Documents and Settings\All Users\Application Data\McAfee\Common Framework\mcscript.log (384749)

  • 문제

    명령줄에서 수행되는 자동 업데이트 작업은 진행률 대화 상자를 표시합니다. 설치가 성공적으로 완료되었지만 명령줄에서 setup.exe /q RUNAUTOUPDATESILENTLY=TRUE를 실행할 때 업데이트 진행 중 대화 상자가 나타납니다. (399265)

  • 문제

    명령줄에서 setup.exe CMASOURCEDIR="<drive>:\Documents and Settings\<username>\Desktop\" 명령을 실행하여 제품을 설치하고 데스크톱에서 Sitelist.xml 파일을 가져올 때 설치는 성공적으로 완료되지만 Sitelist.xml 파일을 가져오지 못합니다.

    해결 방법: ePolicy Orchestrator 에이전트 4.0에서 생성된 Sitelist.xml 파일을 사용합니다. 만약 McAfee 에이전트 4.0 설치로 이전 ePolicy Orchestrator 에이전트 3.6.1의 설치가 업그레이드되었다면 가져올 수 있는 올바른 Sitelist.xml 파일이 생성됩니다. (419467)

원격 콘솔

  • 문제

    원격 콘솔 기능을 사용하여 Windows Vista 시스템 이상에 연결할 경우 원격 시스템에 대한 액세스 보호, 버퍼 오버플로 및 검역 관리자 대화 상자를 열 수 없고 오류가 나타나지 않습니다. 이는 Windows Vista 이상 시스템에서는 원격 콘솔 기능에 필요한 원격 레지스트리 서비스를 자동으로 시작하지 않기 때문입니다.

    해결 방법: 원격 콘솔 기능을 시작하기 전에 다음을 사용하여 원격 시스템에서 원격 레지스트리 서비스를 수동으로 시작합니다.
    • 시작 | 실행을 클릭하고 services.msc를 입력합니다. 서비스 대화 상자가 나타납니다.
    • 서비스(로컬) | 이름 열에서 원격 레지스트리를 마우스 오른쪽 버튼으로 클릭하고 시작을 클릭합니다.
    (581439)
  • 문제

    Windows Server 2008이 설치된 시스템에서 원격 콘솔 기능을 사용하여 액세스 보호 속성을 열 경우 연결을 여는 데 몇 분이 걸릴 수 있습니다. (392825)

기타

  • 문제

    CheckPoint pireg.exe 유틸리티를 실행할 때 다음과 같은 오류 메시지가 표시됩니다.

    mcavscv.dll이 존재하지 않습니다.

    해결 방법: 이 오류가 표시되지 않게 하려면 Microsoft Visual C++ 2008 재배포 가능 패키지를 설치합니다. (629324)

  • 문제

    사용자 계정 제어(UAC) 창 프롬프트에서 특정 설정을 변경하고 아니오를 클릭해도 변경 사항이 여전히 적용됩니다. 이 문제는 Microsoft Outlook 2010을 사용하는 다음과 같은 시스템에만 영향을 줍니다.

    • Microsoft Windows 7 또는 Microsoft Windows Server 2008(32비트 또는 64비트) 시스템
    • UAC를 활성화한 시스템

    특히 이러한 환경에서는 다음과 같이 됩니다.

    1. Microsoft Outlook 2010에서 주문형 전자 메일 검색 등록 정보를 여는 경우 보고서 탭 설정을 기본값에서 다른 설정(예: 세션 설정 활성화)으로 변경하고 적용 또는 확인을 클릭합니다.
    2. Windows UAC 창에서 다음 프로그램이 이 컴퓨터를 변경할 수 있도록 허용하시겠습니까?라고 물을 때 아니오를 클릭하여 변경을 무시해도 변경이 수행됩니다.

    해결 방법: 없음. Windows UAC의 문제이며 해결 방법이 없습니다. (627058)

  • 문제

    온액세스 검색을 비활성화해도 ScriptScan이 비활성화되지 않습니다. VirusScan Enterprise 8.7 이하에서는 온액세스 검색을 비활성화한 경우 ScriptScan 기능도 비활성화됩니다. VirusScan Enterprise 8.8에서는 온액세스 검색을 비활성화한 경우 ScriptScan이 계속 실행됩니다.

    해결 방법: 이러한 사용자 인터페이스 중 하나를 사용하여 VirusScan Enterprise 8.8 ScriptScan 기능을 비활성화합니다.
    • ePolicy Orchestrator 4.5 또는 4.6
      1. 메뉴 | 정책 | 정책 카탈로그를 클릭한 다음 제품 목록에서 VirusScan Enterprise 8.8.0을 선택합니다. 범주 목록은 VirusScan Enterprise 8.8.0에 대한 정책 범주를 표시합니다.
      2. 범주 목록에서 온액세스 일반 정책을 선택하고 작업 열에서 설정 편집을 클릭하여 정책 구성 페이지를 엽니다.
      3. 설정 목록에서 워크스테이션 또는 서버를 선택합니다.
      4. 온액세스 일반 정책 페이지에서 ScriptScan 탭을 클릭합니다.
      5. 스크립트 검색 활성화 확인란을 선택 취소합니다.
      6. 저장을 클릭하여 구성 변경을 캡처합니다.
    • ePolicy Orchestrator 4.0
      1. 시스템 | 정책 카탈로그를 클릭한 다음 제품 목록에서 VirusScan Enterprise 8.8.0을 선택합니다. 범주 목록은 VirusScan Enterprise 8.8.0에 대한 정책 범주를 표시합니다.
      2. 범주 목록에서 온액세스 일반 정책을 선택하고 작업 열에서 설정 편집을 클릭하여 정책 구성 페이지를 엽니다.
      3. 설정 목록에서 워크스테이션 또는 서버를 선택합니다.
      4. 온액세스 일반 정책 페이지에서 ScriptScan 탭을 클릭합니다.
      5. 스크립트 검색 활성화 확인란을 선택 취소합니다.
      6. 저장을 클릭하여 구성 변경을 캡처합니다.
