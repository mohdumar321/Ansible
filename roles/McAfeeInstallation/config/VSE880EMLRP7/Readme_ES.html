Acerca de este documento

Gracias por utilizar el software VirusScan Enterprise. Este documento contiene información importante acerca de esta versión. Le recomendamos encarecidamente que lea el documento entero.

ATENCIÓN: No damos soporte a la actualización automática de versiones anteriores del software. Si desea actualizar el software a una versión de producción, primero deberá desinstalar la versión existente del software.

La versión VirusScan Enterprise 8.8 utiliza lo siguiente:

  • Versión del motor: 5400.1158
  • Definición de detección (DAT): esta versión del software VirusScan Enterprise descarga automáticamente la última versión de los archivos DAT cuando se instala el software. Anteriormente, los archivos DAT se incluían en el ejecutable de instalación.

Nuevas funciones

Funciones nuevas y actualizadas de la versión del software VirusScan Enterprise 8.8.

Mejoras del rendimiento

Esta versión reduce lo siguiente:
  • Tiempo de análisis en tiempo real
  • Tiempo de análisis bajo demanda
  • Tiempo de inicio del sistema

Almacena en caché los archivos analizados anteriormente en una ubicación común para reducir el análisis duplicado de archivos y mejorar el rendimiento.

Compatibilidad con Windows Office 2010

Esta versión es compatible con todas las aplicaciones de Windows Office 2010 y, en concreto, agrega funciones de análisis en tiempo real del correo electrónico y los archivos adjuntos de Microsoft Outlook 2010.

Compatibilidad con las exclusiones de ScriptScan

Se pueden agregar fácilmente exclusiones de ScriptScan (direcciones URL de secuencias de comandos de confianza de sitios web conocidos seguros) para reducir los análisis y mejorar el rendimiento. Esta función está disponible en ePolicy Orchestrator 4.0 o posterior, y en la Consola de VirusScan. Anteriormente, estas direcciones URL excluidas requerían la edición manual del registro.

Migración de directivas y eventos en ePolicy Orchestrator

  • Problema

    Es posible que la herramienta de migración de ePolicy Orchestrator no pueda migrar las directivas y tareas existentes de un servidor ePolicy Orchestrator que utilice la autenticación de Windows NT para el acceso a ePolicy Orchestrator. En algunos casos, la herramienta de migración de ePolicy Orchestrator debe pasar la contraseña de administrador de SQL como parámetro para la herramienta de migración de ePolicy Orchestrator.

    Solución: ejecute el comando ePOPolicyMigration.exe desde la línea de comandos y agregue el modificador /PASSWORD y la contraseña de administrador de SQL. Por ejemplo, escriba: [ruta]\epopolicymigration.exe /password=contraseña_admin_SQL

    Para obtener información acerca del uso del modificador /PASSWORD de ePolicy Orchestrator en la línea de comandos, consulte la Guía de instalación de McAfee VirusScan Enterprise 8.8.0. (611047)

Plataformas y productos compatibles

  • Esta versión de VirusScan Enterprise es compatible con Lotus Notes, versión 7.0.x y 8.0.x – 8.5.1. Consulte la Guía de instalación de VirusScan Enterprise 8.8 para obtener información acerca de las aplicaciones compatibles.

Compatibilidad con otros productos

    En la interfaz de usuario de ePolicy Orchestrator, la opción Solicitar acción no se encuentra en las listas Realizar esta acción primero. Por ejemplo, si el usuario está configurando una tarea de análisis bajo demanda a través de la interfaz de usuario de ePolicy Orchestrator y:
    • Hace clic en Árbol del sistema | Tareas de cliente | Tareas del análisis bajo demanda para VSE 8.8.0
    • Hace clic en las fichas Editar configuración, Configuración y Acciones

    La selección de la tarea Solicitar acción falta en las listas Realizar esta acción primero de este cuadro de diálogo.

    La selección de la tarea Solicitar acción está disponible en la interfaz de usuario de la Consola de VirusScan. Por ejemplo:
    • Si hace doble clic en una tarea de análisis bajo demanda, haga clic en la ficha Acciones del cuadro de diálogo Propiedades del análisis bajo demanda.
    • Puede seleccionar la tarea Solicitar acción en las listas Realizar esta acción primero.

    Solución: no existe. Debido a que se trata de una tarea de análisis gestionada de ePolicy Orchestrator, no hay ninguna interfaz a través de la cual el usuario pueda ver cuándo se está ejecutando una tarea de análisis bajo demanda. Por lo tanto, no hay ninguna configuración para solicitar al usuario una acción. (590803)

  • Problema

    Después de actualizar correctamente de la versión VirusScan Enterprise 8.7 a la versión VirusScan Enterprise 8.8, el cuadro de diálogo Detalles del sistema de ePolicy Orchestrator 4.5 no incluye VirusScan Enterprise 8.8 en la fila de Productos instalados. (620509)

  • Problema
    Después de instalar VirusScan Enterprise 8.8 en un sistema donde está instalado McAfee Network Access Control 3.0, podría producirse un error que impida la identificación de los servicios de McAfee. Es posible que observe los siguientes síntomas:
    • Se registra un mensaje de tipo el controlador especificado no es válido cuando lleva a cabo una tarea de actualización antes de reiniciar.
    • Se registra un mensaje de tipo no se pudo cargar el controlador para uno o ambos productos.
    • El registro de eventos de la aplicación de sistema local contiene el ID de evento 5004.

    Solución: desinstale el producto defectuoso, reinicie el sistema y vuelva a instalar el producto.

  • Problema

    Los controladores NVIDIA® podrían provocar problemas de rendimiento o un fallo en el sistema. VirusScan Enterprise 8.8 podría ejecutarse a un 100% de utilización de la CPU o provocar un fallo en el sistema. Consulte los artículos KB51688 y KB52367 de KnowledgeBase para obtener más información sobre este problema. (388060)

  • Problema

    Cuando se ejecuta el instalador de VirusScan Enterprise 8.8, no se eliminan por completo la estructura de directorios y las entradas de la clave de Registro de las versiones 7.0 y 7.1 del programa CA Anti-Virus.

    Solución: no existe. Se trata de un problema relacionado con el programa de desinstalación de CA Anti-Virus. (625703)

  • Problema

    En sistemas cliente independientes, VirusScan Enterprise 8.8 no puede completar la actualización inicial del producto cuando Panda Anti-Virus 2008 también está instalado.

    Si se ejecuta VSESetup.exe y se selecciona la opción No para no eliminar el software Panda Anti-Virus 2008, es posible que en la última fase (en la cual VirusScan Enterprise 8.8 realiza la actualización del producto) el cuadro de diálogo de actualización se quede "colgado" durante 20 minutos o más.

    Solución: si ejecuta VSESetup.exe, seleccione la opción Sí para eliminar el software Panda Anti-Virus 2008. (626009)

  • Problema

    Cuando, en un cliente gestionado de ePolicy Orchestrator, se ejecuta VSESetup.exe y se selecciona la opción que permite eliminar el software antivirus detectado, algunos archivos, directorios y claves de registro de Symantec no se eliminan del sistema.

    Solución: no existe. (629265)

Actualización

  • Problema
    Es posible que el registro de actualización informe de un "Error en la actualización" y que no indique ningún otro error. La función de actualización para parches, DAT y motores necesita suficiente espacio libre y memoria disponible para ampliar los paquetes de actualización. Debe disponer de los siguientes requisitos de espacio mínimo para una actualización correcta:
    • 125 MB de espacio libre en el disco duro
    • 250 MB de espacio libre total en el disco duro y la memoria
      NOTA: La memoria virtual de Windows también se puede utilizar para crear espacio adicional en la memoria, pero ese total ha de ser igual a 250 MB de espacio libre.

    Este requisito de disco duro y memoria es particularmente importante para sistemas incrustados y virtuales donde a menudo se ejecutan los requisitos mínimos del sistema. (625749)

  • Problema

    Los fallos de acceso al repositorio no se registran en el registro de actualización de VirusScan Enterprise 8.8, pero sí en el registro de McAfee Agent. La ubicación predeterminada del registro de McAfee Agent es: <unidad>:\Documents and Settings\All Users\Application Data\McAfee\Common Framework\mcscript.log. (384749)

  • Problema

    Las tareas de actualización silenciosa llevadas a cabo desde la línea de comandos siguen mostrando el cuadro de diálogo de progreso. La instalación finaliza correctamente, pero aparece el cuadro de diálogo Actualización en curso cuando se ejecuta setup.exe /q RUNAUTOUPDATESILENTLY=TRUE desde la línea de comandos. (399265)

  • Problema

    Al ejecutar el comando setup.exe CMASOURCEDIR="<unidad>:\Documents and Settings\<nombreDeUsuario>\Desktop\" desde la línea de comandos para instalar el producto e importar el archivo Sitelist.xml desde el escritorio, la instalación finaliza correctamente, pero no se importa el archivo Sitelist.xml.

    Solución: utilice el archivo Sitelist.xml creado por el agente de ePolicy Orchestrator 4.0. Si la instalación de McAfee Agent 4.0 actualizó una instalación anterior del agente de ePolicy Orchestrator 3.6.1, se crea un archivo Sitelist.xml correcto que se puede importar. (419467)

Consola remota

  • Problema

    Al utilizar la función de consola remota para conectarse a sistemas Windows Vista o posteriores, no puede abrir los cuadros de diálogo de protección de acceso, desbordamiento de búfer y gestor de cuarentena de los sistemas remotos y no aparece ningún error. Esto se debe a que los sistemas Windows Vista y posteriores no inician automáticamente el servicio Registro remoto requerido por la función de consola remota.

    Solución: antes de iniciar la función de consola remota, inicie manualmente el servicio Registro remoto en el sistema remoto. Para ello, siga estos pasos:
    • Haga clic en Inicio | Ejecutar y escriba services.msc. Aparece el cuadro de diálogo Servicios.
    • Haga clic en el botón secundario en Registro remoto, en la columna Servicios (locales) | Nombre, y haga clic en Iniciar.
    (581439)
  • Problema

    Al utilizar la función de consola remota para abrir las propiedades de Protección de acceso en un sistema con Windows Server 2008, puede que se tarden varios minutos en abrir la conexión. (392825)

Varios

  • Problema

    Al ejecutar la utilidad pireg.exe de CheckPoint, aparece el siguiente mensaje de error:

    mcavscv.dll no existe

    Solución: para evitar que aparezca este error, instale Microsoft Visual C++ 2008 Redistributable Package. (629324)

  • Problema

    Cuando realiza determinados cambios de configuración en la ventana del símbolo del sistema del control de cuentas de usuario (UAC) y hace clic en No, los cambios se ejecutan de todos modos. Este problema sólo afecta a los siguientes sistemas que utilizan Microsoft Outlook 2010:

    • Microsoft Windows 7 o Microsoft Windows Server 2008 (32 o 64 bits)
    • Sistemas con UAC activado

    Específicamente, en los entornos siguientes:

    1. Cuando abre Propiedades del análisis de correo electrónico bajo demanda desde Microsoft Outlook 2010, modifica la configuración predeterminada de la ficha Informes (por ejemplo, activa la Configuración de sesión) y hace clic en Aplicar o Aceptar.
    2. Cuando la ventana de UAC de Windows le pregunta ¿Desea permitir que este programa realice cambios en el equipo? y hace clic en No para descartar el cambio, éste se ejecuta de todos modos.

    Solución: no existe. Se trata de un problema relacionado con el UAC de Windows y no tiene solución. (627058)

  • Problema

    Al desactivar el análisis en tiempo real, no se desactiva ScriptScan. Con VirusScan Enterprise 8.7 y anteriores, si se desactivaba el análisis en tiempo real también se desactivaba la función ScriptScan. Con VirusScan Enterprise 8.8, si se desactiva el análisis en tiempo real, ScriptScan continúa ejecutándose.

    Solución: utilice una de estas interfaces de usuario para desactivar la función ScriptScan de VirusScan Enterprise 8.8:
    • ePolicy Orchestrator 4.5 o 4.6
      1. Haga clic en Menú | Directiva | Catálogo de directivas y, a continuación, en la lista de productos seleccione VirusScan Enterprise 8.8.0. La lista de categorías muestra las categorías de directivas para VirusScan Enterprise 8.8.0.
      2. En la lista de categorías, seleccione Directivas generales en tiempo real y, en la columna Acciones, haga clic en Editar configuración para abrir la página de configuración de directivas.
      3. En la lista Configuración para, seleccione Estación de trabajo o Servidor.
      4. En la página Directivas generales en tiempo real, haga clic en la ficha ScriptScan.
      5. Anule la selección de la casilla de verificación Activar el análisis de secuencias de comandos.
      6. Haga clic en Guardar para capturar el cambio de configuración.
    • ePolicy Orchestrator 4.0
      1. Haga clic en Sistemas | Catálogo de directivas y, a continuación, en la lista de productos seleccione VirusScan Enterprise 8.8.0. La lista de categorías muestra las categorías de directivas para VirusScan Enterprise 8.8.0.
      2. En la lista de categorías, seleccione Directivas generales en tiempo real y, en la columna Acciones, haga clic en Editar configuración para abrir la página de configuración de directivas.
      3. En la lista Configuración para, seleccione Estación de trabajo o Servidor.
      4. En la página Directivas generales en tiempo real, haga clic en la ficha ScriptScan.
      5. Anule la selección de la casilla de verificación Activar el análisis de secuencias de comandos.
      6. Haga clic en Guardar para capturar el cambio de configuración.
    • Consola de VirusScan
      1. En la lista Tarea, haga clic con el botón secundario en Analizador en tiempo real y, a continuación, haga clic en Propiedades para abrir el cuadro de diálogo.
      2. En la página Propiedades del análisis en tiempo real, haga clic en Configuración general, en el panel izquierdo, y haga clic en la ficha ScriptScan.
      3. Anule la selección de la casilla de verificación Activar el análisis de secuencias de comandos.
      4. Haga clic en Aceptar para capturar el cambio de configuración.
  • Problema

    Con Microsoft Outlook 2010, si desea llevar a cabo un análisis bajo demanda en una carpeta seleccionada, sólo se analiza un elemento en la carpeta.

    NOTA: Este problema no se produce en Microsoft Outlook 2007.

    Solución: para llevar a cabo un análisis bajo demanda en una carpeta entera de Microsoft Outlook 2010:

    1. Seleccione el nombre de la carpeta que desea analizar.
    2. Abra la carpeta y anule la selección del elemento único resaltado con CTRL + clic izquierdo en el elemento.
    3. Ejecute el análisis bajo demanda y se analizarán todos los elementos de la carpeta.

    (624659)

  • Problema

    Con Microsoft Windows 2000 SP4, si intenta editar la lista de repositorios de AutoUpdate con el comando Herramientas | Editar lista de repositorios de AutoUpdate en la Consola de VirusScan, podría aparecer el siguiente error:

    No se ha podido leer la lista de repositorios. Error = fffffffc

    Si aparece el error, significa que falta el archivo SiteList.xml en la siguiente ruta:

    C:\Documents and Settings\All Users\Application Data\McAfee\Common Framework

    Solución: copie un archivo SiteList.xml del paquete de instalación de McAfee Agent 4.0, u otra ubicación, en la ruta. (568699)

  • Problema

    Cuando configure una unidad para que quede excluida del análisis en tiempo real, utilice una ruta UNC, no una unidad de red asignada. Por ejemplo, si escribe Z:, para la unidad de red asignada en el cuadro de diálogo Establecer exclusiones, los archivos de la unidad asignada no se excluyen del análisis en tiempo real.

    Solución: para excluir una unidad de red del análisis en tiempo real, utilice una ruta UNC, no una unidad de red asignada. (546966)

  • Problema
    El uso del juego de caracteres ampliado al agregar Tipos de archivos predeterminados y adicionales no funciona como está previsto al configurar Propiedades del análisis de correo electrónico durante la recepción o Análisis bajo demanda de Outlook. Por ejemplo, si agrega el tipo de archivo ÃbÃ a los tipos de archivos adicionales especificados por el usuario para analizar, lo normal sería que sólo se analizaran archivos del tipo "xxxxx.ÃbÃ". Sin embargo, se analizan los siguientes archivos:
    • xxxxx.ÃbÃ
    • yyyyy.Ãbc
    Éste es el comportamiento esperado al utilizar el juego de caracteres ampliado.
    NOTA: Esto ocurre únicamente en algunos sistemas Windows de 64 bits que utilizan Windows Outlook 2010 x64 Professional Edition.

    Para analizar sólo el tipo de archivo ÃbÃ, configure una exclusión con los caracteres comodín ? o *. Por ejemplo, configure una exclusión con Ãb? o Ã*.

    (586207)

  • Problema

    El analizador de correo electrónico de Lotus Notes McAfee no utiliza el Gestor de cuarentena para poner en cuarentena las amenazas detectadas. Por tanto, si configura la primera acción del analizador de correo electrónico como Limpiar datos adjuntos o Eliminar datos adjuntos, la versión original de la amenaza detectada ya no estará disponible para su recuperación o restauración. Si desea que la versión original de la amenaza detectada esté disponible después de realizar la acción en la detección, McAfee recomienda establecer la primera acción como Mover los datos adjuntos a una carpeta y especificar una carpeta de cuarentena. Cuando se activa esta acción, la amenaza detectada originalmente se mueve a la carpeta de cuarentena especificada con la extensión .MCM incluida en la detección.

    Si activa la opción Mover los datos adjuntos a una carpeta en sistemas cliente, debe tener siempre activado el analizador en tiempo real para que los usuarios no puedan acceder a estos archivos involuntariamente. (430862)

  • Problema

    Al ejecutar VirusScan Enterprise 8.7i en un sistema con Microsoft Windows Server 2008, es posible que el analizador en tiempo real no pueda eliminar un archivo detectado de una carpeta de red compartida. La capacidad del analizador en tiempo real de eliminar un archivo detectado no se puede garantizar en los sistemas de archivos de red. En este caso, si el archivo detectado no se elimina, se elimina el contenido del archivo y el tamaño del archivo restante es cero. (431158)

Documentación

  • Problema

    Después de la publicación de la Guía del producto del software McAfee VirusScan Enterprise 8.8 y la Ayuda correspondiente, se modificó la información de la sección Configuración general.

    En cada una de las consolas de interfaz de usuario se ha modificado el siguiente paso para incluir una nota:

    En la ficha ScriptScan, active ScriptScan y configure los procesos o URL que desee excluir del análisis.
    NOTA: Al desactivar el análisis en tiempo real, no se desactiva ScriptScan. Con VirusScan Enterprise 8.7 y anteriores, al desactivar el análisis en tiempo real se desactivaba la función ScriptScan. Con VirusScan Enterprise 8.8, al desactivar el análisis en tiempo real, no se desactiva ScriptScan. Para desactivar ScriptScan, anule la selección de la casilla de verificación Activar el análisis de secuencias de comandos.
  • Problema

    Después de la publicación de la Ayuda y la Guía del producto de McAfee VirusScan Enterprise 8.8, se modificó la información incluida en la Ayuda de Directivas de análisis en tiempo real relativa a la ficha General.

    En la opción Análisis se agregó la información siguiente:

    Instaladores de confianza: especifique si se analizan en tiempo real los archivos MSI (instalados por msiexec.exe y firmados por McAfee o Microsoft) o los archivos del servicio Instalador de confianza de Windows:
    • No seleccionado: no analizar los archivos instalados por MSI o el servicio Instalador de confianza de Windows. Predeterminado = Desactivado
      NOTA: Si no se selecciona, se mejora el rendimiento de los instaladores de aplicaciones de Microsoft de gran tamaño.
    • Seleccionado: analizar archivos instalados por MSI o el servicio Instalador de confianza de Windows.
      NOTA: No se incluye el análisis de archivos fusionados o archivos MSI de plantillas que formen parte de una instalación de línea de comandos y que se encuentren en una lista blanca.
