Об этом документе

Благодарим за использование программного обеспечения VirusScan Enterprise. Данный документ содержит важную информацию об этом выпуске. Настоятельно рекомендуется прочитать этот документ полностью.

ОСТОРОЖНО: Автоматическое обновление предварительной версии программного обеспечения не поддерживается. Чтобы обновить программное обеспечение до серийной версии, необходимо сначала удалить существующую версию.

В версии VirusScan Enterprise 8.8 используются следующие компоненты.

  • Версия механизма сканирования: 5400.1158
  • Файлы определения обнаружения (DAT): в данной версии VirusScan Enterprise последние версии файлов DAT загружаются при установке программного обеспечения. Раньше файлы DAT были включены в исполняемый файл установки.

Новые функции

Новые и обновленные функции в версии программного обеспечения VirusScan Enterprise 8.8.

Повышение быстродействия

В данной версии программного обеспечения сокращено:
  • время проверки при доступе;
  • время проверки по требованию;
  • время запуска системы.

Проверенные файлы кэшируются в общей папке и не сканируются повторно, в результате чего повышается быстродействие.

Поддержка Windows Office 2010

Данная версия поддерживает приложения Windows Office 2010, а именно, появилась возможность проверки при доступе электронной почты и вложенных файлов Microsoft Outlook 2010.

Поддержка исключений ScriptScan

Для сокращения объема проверки и повышения быстродействия можно легко добавить исключения ScriptScan, URL-адресов доверенных сценариев с известных безопасных веб-сайтов. Данная функция доступна в ePolicy Orchestrator 4.0 или более поздних версиях, а также в Консоли VirusScan. Ранее для исключения данных URL-адресов требовалось изменение реестра вручную.

Миграция политик и событий в ePolicy Orchestrator

  • Проблема

    Инструмент миграции ePolicy Orchestrator может дать сбой при попытке миграции существующих политик и задач с сервера ePolicy Orchestrator, в котором используется проверка подлинности Windows NT для доступа к ePolicy Orchestrator. В отдельных случаях для работы инструмента миграции ePolicy Orchestrator необходимо передать пароль администратора SQL в качестве параметра для инструмента миграции ePolicy Orchestrator.

    Обход: из командной строки выполните команду ePOPolicyMigration.exe с параметром /PASSWORD и паролем администратора SQL. Например, введите: [путь]\epopolicymigration.exe /password=sql_admin_password

    Для получения сведений об использовании параметра ePolicy Orchestrator /PASSWORD в командной строке см. Руководство по установке McAfee VirusScan Enterprise 8.8.0. (611047)

Совместимость с другими продуктами

  • Проблема
    В интерфейсе пользователя ePolicy Orchestrator отсутствует задача Запрос на действие в списке Выполнить это действие первым. Например, при настройке задачи проверки по требованию с помощью интерфейса пользователя ePolicy Orchestrator выполняется следующая процедура:
    • пользователь выбирает Дерево систем | Задачи клиента | Задачи проверки по требованию для VSE 8.8.0;
    • затем Изменить параметры, Конфигурация и вкладку Действия.

    Задача Запрос на действие отсутствует в обоих списках Выполнить это действие первым данного диалогового окна.

    Задача Запрос на действие доступна в интерфейсе пользователя Консоли VirusScan. Пример.
    • Если дважды щелкнуть задачу проверки по требованию, откроется диалоговое окно свойств проверки по требованию, в котором можно перейти на вкладку Действия.
    • Задачу Запрос на действие можно выбрать в списках Выполнить это действие первым.

    Обход: нет. Поскольку эта задача проверки осуществляется ePolicy Orchestrator, ход ее выполнения не отображается в интерфейсе пользователя. Поэтому настройка запроса на выполнение действия отсутствует. (590803)

Обновление

  • Проблема
    В журнале обновления отображается сообщение «Ошибка обновления» без указания других ошибок. Для развертывания пакетов обновлений — исправлений, файлов DAT и механизмов сканирования — функции обновления необходим определенный объем свободного пространства на диске и в памяти. Для успешного обновления необходимо удовлетворение следующих требований к минимальному свободному пространству на диске:
    • 125 МБ свободного пространства на жестком диске;
    • 250 МБ общего пространства на жестком диске и в памяти.
      ПРИМЕЧАНИЕ: Для создания дополнительного пространства в памяти можно также использовать виртуальную память Windows, при этом общий объем свободного пространства должен составлять 250 МБ.

    Соблюдение данных требований к объему жесткого диска и памяти особенно важно для работы встроенных и виртуальных систем с минимальными системными требованиями. (625749)

  • Проблема

    Ошибка доступа к хранилищу регистрируется не в журнале обновления VirusScan Enterprise 8.8, а в журнале агента McAfee Agent. Местоположение журнала агента McAfee Agent по умолчанию: <диск>:\Documents and Settings\All Users\Application Data\McAfee\Common Framework\mcscript.log. (384749)

  • Проблема

    Фоновые задачи обновления, выполняемые из командной строки, по-прежнему отображаются в диалоговом окне хода выполнения. После успешной установки появляется диалоговое окно Выполняется обновление при запуске команды setup.exe /q RUNAUTOUPDATESILENTLY=TRUE из командной строки. (399265)

  • Проблема

    При запуске команды setup.exe CMASOURCEDIR="<диск>:\Documents and Settings\<имя_пользователя>\Рабочий стол\" из командной строки для установки продукта и импорта файла Sitelist.xml с рабочего стола установка выполняется успешно, а импорт файла Sitelist.xml завершается ошибкой.

    Обход: используйте файл Sitelist.xml, созданный агентом ePolicy Orchestrator Agent 4.0. Если установка McAfee Agent 4.0 выполнялась для обновления предыдущей версии агента ePolicy Orchestrator Agent 3.6.1, должен быть создан правильный файл Sitelist.xml, доступный для импорта. (419467)

Удаленная консоль

  • Проблема

    При использовании удаленной консоли для подключения к системам Windows Vista или более поздним версиям не удается открыть диалоговые окна «Защита доступа», «Переполнение буфера» и «Quarantine Manager» для удаленных систем. При этом сообщения об ошибках не отображаются. Это объясняется тем, что в Windows Vista и более поздних версиях системы не запускается автоматически служба удаленного реестра, необходимая для работы удаленной консоли.

    Обход: перед запуском удаленной консоли вручную запустите службу удаленного реестра в удаленной системе, выполнив описанные ниже действия.
    • Щелкните Пуск | Выполнить и введите services.msc. Откроется диалоговое окно «Службы».
    • Щелкните Удаленный реестр в столбце Службы (локальные) | Имя, затем щелкните Запустить.
    (581439)
  • Проблема

    При открытии с помощью удаленной консоли окна свойств Защита доступа в Windows Server 2008 установка подключения может занимать несколько минут. (392825)

Прочие проблемы

  • Проблема

    При запуске служебной программы CheckPoint pireg.exe отображается следующее сообщение об ошибке:

    mcavscv.dll не существует

    Обход: чтобы эта ошибка больше не отображалась, установите распространяемый пакет Microsoft Visual C++ 2008. (629324)

  • Проблема

    При выборе параметра Нет в ответ на запрос сохранения изменений в окне управления учетными записями пользователя (UAC) изменения все равно сохраняются. Данная проблема наблюдается только в следующих системах с установленным приложением Microsoft Outlook 2010:

    • система Microsoft Windows 7 или Microsoft Windows Server 2008 (32- или 64-разрядная версия);
    • системы с функцией управления учетными записями пользователя (UAC).

    В частности, проблема наблюдается в следующих средах.

    1. Если открыть диалоговое окно Свойства сканирования электронной почты по требованию из Microsoft Outlook 2010, изменить параметры по умолчанию на вкладке Отчеты, например установить параметр Настройки сеанса и щелкнуть Применить или OK.
    2. При отображении в Windows запроса UAC Разрешить следующей программе внести изменения на этом компьютере? даже если щелкнуть Нет для отмены изменения, изменение все равно будет выполнено.

    Обход: нет. Данная проблема связана с функцией UAC в Windows и решение для нее отсутствует. (627058)

  • Проблема

    При отключении проверки при доступе ScriptScan не отключается. В VirusScan Enterprise 8.7 и более ранних версиях программного обеспечения при отключении проверки при доступе также отключалась функция ScriptScan. В VirusScan Enterprise 8.8 при отключении проверки при доступе функция ScriptScan продолжает работать.

    Обход: используйте один из следующих интерфейсов пользователя для отключения функции VirusScan Enterprise 8.8 ScriptScan.
    • ePolicy Orchestrator 4.5 или 4.6
      1. Щелкните Меню | Политика | Каталог политик, затем в списке продуктов выберите VirusScan Enterprise 8.8.0. В списке «Категория» отобразятся категории политик для VirusScan Enterprise 8.8.0.
      2. В списке «Категория» выберите Общие политики проверки при доступе, а в столбце «Действия» щелкните Изменить параметры, чтобы открыть страницу настройки политики.
      3. В списке «Настройки» выберите Рабочая станция или Сервер.
      4. На странице «Общие политики проверки при доступе» щелкните вкладку ScriptScan.
      5. Снимите флажок Включить сканирование сценариев.
      6. Щелкните Сохранить, чтобы сохранить изменения настроек.
    • ePolicy Orchestrator 4.0
      1. Щелкните Системы | Каталог политик, затем в списке продуктов выберите VirusScan Enterprise 8.8.0. В списке «Категория» отобразятся категории политик для VirusScan Enterprise 8.8.0.
      2. В списке «Категория» выберите Общие политики проверки при доступе, а в столбце «Действия» щелкните Изменить параметры, чтобы открыть страницу настройки политики.
      3. В списке «Настройки» выберите Рабочая станция или Сервер.
      4. На странице «Общие политики проверки при доступе» щелкните вкладку ScriptScan.
      5. Снимите флажок Включить сканирование сценариев.
      6. Щелкните Сохранить, чтобы сохранить изменения настроек.
    • Консоли VirusScan
      1. В списке «Задача» щелкните правой кнопкой мыши Сканер при доступе, а затем щелкните Свойства, чтобы открыть диалоговое окно.
      2. На странице «Свойства проверки при доступе» щелкните Общие параметры в левой панели, а затем щелкните вкладку ScriptScan.
      3. Снимите флажок Включить сканирование сценариев.
      4. Щелкните OK, чтобы сохранить изменения настроек.

Однако проверяются оба файла:
    • xxxxx.aaa

  • Проблема

    Сканер электронной почты McAfee для Lotus Notes не использует Quarantine Manager для помещения обнаруженных угроз в карантин. Соответственно, если в сканере электронной почты первым действием указано Очистить вложения или Удалить вложения, исходная версия обнаруженной угрозы становится недоступной для восстановления. Если необходимо, чтобы исходная версия обнаруженной угрозы была доступна после выполнения действия над обнаруженной угрозой, McAfee рекомендует задать в качестве первого действия Переместить вложения в папку и указать папку карантина. После задания данного действия исходная обнаруженная угроза перемещается в указанную папку карантина, при этом к обнаружению добавляется расширение .MCM.

    Если на клиентских системах выбран параметр Переместить вложения в папку, сканер при доступе должен быть всегда включен, чтобы пользователь случайно не открыл эти файлы. (430862)

  • Проблема

    При использовании VirusScan Enterprise 8.7i в Microsoft Windows Server 2008 сканер при доступе может не удалять обнаруженный файл из общей сетевой папки. Корректное удаление обнаруженных файлов сканером при доступе не гарантируется в сетевых файловых системах. В этом случае если обнаруженный файл не удаляется, его содержимое очищается и оставшийся размер файла становится равным нулю. (431158)

Документация

  • Проблема

    После выпуска руководства по эксплуатации программного обеспечения McAfee VirusScan Enterprise 8.8 и Справки информация в разделе Настройка общих параметров была изменена.

    В процедуру использования каждой консоли интерфейса пользователя было добавлено следующее примечание к одному из шагов.

    На вкладке ScriptScan включите ScriptScan и настройте процессы или URL-адреса, которые требуется исключить из сканирования.
    ПРИМЕЧАНИЕ: При отключении проверки при доступе ScriptScan не отключается. В VirusScan Enterprise 8.7 и более ранних версиях программного обеспечения при отключении проверки при доступе также отключалась функция ScriptScan. В VirusScan Enterprise 8.8 при отключении проверки при доступе функция ScriptScan не отключается. Чтобы отключить функцию ScriptScan, снимите флажок Включить сканирование сценариев.
  • Проблема

    После выпуска Руководства по эксплуатации McAfee VirusScan Enterprise 8.8 и Справки информация в разделе справки «Политики проверки при доступе» для вкладки «Общие» была изменена.

    Для параметра сканирования была добавлена следующая информация.

    Доверенные установщики: указывается необходимость проверки при доступе файлов MSI, установленных с помощью msiexec.exe и подписанных McAfee или Microsoft, а также служебных файлов доверенного установщика Windows.
    • Если параметр не установлен, проверка файлов, установленных MSI или доверенным установщиком Windows, не выполняется. По умолчанию параметр отключен.
      ПРИМЕЧАНИЕ: При выключенном параметре повышается эффективность работы установщиков приложений Microsoft.
    • Если параметр установлен, проверка файлов, установленных MSI или доверенным установщиком Windows, выполняется.
      ПРИМЕЧАНИЕ: Это не распространяется на проверку объединенных файлов или шаблонов MSI, которые используются в процессе установки с помощью командной строки и включены в белый список.
