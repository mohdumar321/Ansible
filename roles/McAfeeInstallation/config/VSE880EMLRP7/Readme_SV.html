Om det här dokumentet

Tack för att du har valt VirusScan Enterprise-programvaran. Det här dokumentet innehåller viktig information om den här programversionen. Vi rekommenderar att du läser igenom hela dokumentet.

VAR FÖRSIKTIG Vi har inte stöd för automatisk uppgradering av förhandsversioner. Om du vill uppgradera till en försäljningsversion måste du först avinstallera den version du har.

I VirusScan Enterprise 8.8 används följande:

  • Motorversion: 5400.1158
  • Definitionsfiler för identifiering (DAT): Med den här versionen av VirusScan Enterprise hämtas automatiskt den senaste versionen av DAT-filerna när programvaran installeras. I tidigare versioner ingick DAT-filerna i den körbara installationsfilen.

Nya funktioner

Nya och uppdaterade funktioner i VirusScan Enterprise 8.8-programutgåvan.

Prestandaförbättringar

Den här programversionen ger snabbare:
  • Genomsökning vid läsning
  • Genomsökning på begäran
  • Systemstart

Tidigare genomsökta filer cachelagras på en gemensam plats för att undvika att dubbla filer genomsöks och för att öka prestandan.

Stöd för Windows Office 2010

Den här utgåvan har stöd för alla Windows Office 2010-program och innehåller en funktion för genomsökning vid läsning av e-post och bifogade filer i Microsoft Outlook 2010.

Stöd för ScriptScan-undantag

Undantag för ScriptScan, webbadresser för betrodda skript från kända säkra webbplatser, kan lätt läggas till för att minska genomsökningsomfånget och förbättra prestandan. Den här funktionen finns i ePolicy Orchestrator 4.0 eller senare, och i VirusScan-konsolen. Tidigare krävdes att de undantagna webbadresserna redigerades manuellt i registret.

Migrering av principer och händelser i ePolicy Orchestrator

  • Problem

    Migreringsverktyget i ePolicy Orchestrator kan ibland inte migrera befintliga principer och aktiviteter från en ePolicy Orchestrator-server som använder Windows NT-autentisering för att få åtkomst till ePolicy Orchestrator. I vissa fall måste migreringsverktyget i ePolicy Orchestrator skicka SQL-administratörslösenordet som parameter för migreringsverktyget i ePolicy Orchestrator.

    Lösning: Kör kommandot ePOPolicyMigration.exe från kommandoraden och lägg till växeln /PASSWORD och SQL-administratörslösenordet. Skriv till exempel: [sökväg]\epopolicymigration.exe /password=sql-adminlösenord

    Information om hur du använder växeln ePolicy Orchestrator /PASSWORD på kommandoraden finns i Installationshandboken för McAfee VirusScan Enterprise 8.8.0. (611047)

Kompatibilitet med andra produkter

  • Problem
    I användargränssnittet för ePolicy Orchestrator saknas åtgärdsvalet Prompt for Action (Begär åtgärd) från listorna Perform this action first (Utför denna åtgärd först). Om du t.ex. konfigurerar en åtgärd för genomsökning på begäran med användargränssnittet i ePolicy Orchestrator och du:
    • klickar på System Tree | Client Tasks | On-Demand Scan Tasks for VSE 8.8.0 (Systemträd | Klientaktiviteter | Genomsökningsåtgärder på begäran för VSE 8.8.0)
    • klickar på flikarna Edit settings (Redigera inställningar), Configuration (Konfiguration) och Actions (Åtgärder)

    Åtgärdsvalet Prompt for Action (Begär åtgärd) saknas från båda listorna Perform this action first (Utför denna åtgärd först) i dialogrutan.

    Åtgärdsvalet Prompt for Action (Begär åtgärd) är tillgängligt via användargränssnittet i VirusScan-konsolen. Till exempel:
    • Om du dubbelklickar på en genomsökningsaktivitet på begäran, klickar du på fliken Actions (Åtgärder) i dialogrutan On-Demand Scan Properties (Egenskaper för genomsökning på begäran).
    • Du kan välja åtgärden Prompt for Action (Begär åtgärd) från listorna Perform this action first (Utför denna åtgärd först).

    Lösning: Ingen. Eftersom det här är en genomsökningsaktivitet som hanteras av ePolicy Orchestrator, finns det inget gränssnitt som användaren kan se när en genomsökningsaktivitet på begäran utförs. Därför finns det ingen konfiguration som frågar användaren om en åtgärd. (590803)

  • Problem

    Efter en lyckad uppgradering från VirusScan Enterprise 8.7 till VirusScan Enterprise 8.8 visar dialogrutan System Details (Systeminformation) i ePolicy Orchestrator 4.5 inte VirusScan Enterprise 8.8 i raden Installed Products (Installerade produkter). (620509)

    När du har installerat VirusScan Enterprise 8.8 på ett system där McAfee Network Access Control 3.0 är installerat, kan det inträffa ett fel som gör att McAfee-tjänsterna inte kan identifieras. Följande symptom kan förekomma:
    • Meddelandet Den angivna drivrutinen är ogiltig loggas om du utför en uppdateringsaktivitet innan du startar om.
    • Meddelandet Drivrutinen kunde inte läsas in loggas för en eller båda produkterna.
    • Programhändelseloggen för det lokala systemet innehåller händelse-ID 5004.

    Lösning: Avinstallera produkten ifråga, starta om systemet och installera produkten på nytt.

  • Problem

    NVIDIA®-drivrutiner kan orsaka problem med prestanda eller systemfel. VirusScan Enterprise 8.8 kan köras med en hundraprocentig belastning av processorn eller orsaka ett systemfel. Mer information om det här problemet finns i KnowledgeBase-artiklarna KB51688 och KB52367. (388060)

  • Problem

    När installationsprogrammet för VirusScan Enterprise 8.8 körs tas inte alla programkatalogstrukturer och registernycklar bort för CA Anti-Virus version 7.0 och 7.1.

    Lösning: Ingen. Det här är ett problem med avinstallationsprogrammet för CA Anti-Virus. (625703)

  • Problem

    På fristående klientsystem slutför inte VirusScan Enterprise 8.8 den inledande produktuppdateringen när Panda Anti-Virus 2008 också är installerat.

    När du kör VSESetup.exe och har valt alternativet Nej, att inte ta bort programmet Panda Anti-Virus 2008, kan dialogrutan med uppdateringen hänga sig under 20 minuter eller längre under det sista stadiet där VirusScan Enterprise 8.8 ska utföra produktuppdateringen.

    Lösning: När du kör VSESetup.exe ska du välja alternativet Ja så att programmet Panda Anti-Virus 2008 tas bort. (626009)

  • Problem

    På en klient som hanteras av ePolicy Orchestrator, finns vissa antivirusfiler, kataloger och registernycklar från Symantec kvar på systemet om du kör VSESetup.exe och väljer alternativet att ta bort identifierade antivirusprogram.

    Lösning: Ingen. (629265)

Uppdatering

  • Problem
    Uppdateringsloggen kan rapportera ett uppdateringsfel utan att något annat fel visas. Uppdateringsfunktionen för korrigeringsfiler, DAT-filer och motorer kräver att det finns tillräckligt med ledigt utrymme och minne för att expandera uppgraderingspaketen. Du måste uppfylla följande minimikrav på ledigt utrymme för att uppdateringen ska kunna genomföras:
    • 125 MB ledigt hårddiskutrymme
    • 250 MB sammanlagt ledigt hårddisk- och minnesutrymme
      OBS! Windows virtuella minne kan också användas för att skapa ytterligare minnesutrymme, men sammantaget måste det finnas minst 250 MB ledigt utrymme.

    Kraven för hårddisk och minne är särskilt viktiga för inbäddade och virtuella system som ofta körs med minimisystemkraven. (625749)

  • Problem

    Fel vid åtkomst till lagringsplats loggas inte i uppdateringsloggen för VirusScan Enterprise 8.8, utan i McAfee Agent-loggen. Standardplatsen för McAfee Agent-loggen är: <enhet>:\Documents and Settings\All Users\Application Data\McAfee\Common Framework\mcscript.log. (384749)

  • Problem

    Obevakade uppdateringar som utförts från kommandoraden visas fortfarande i förloppsdialogrutan. Installationen slutförs utan problem men dialogrutan Uppdatering pågår visas när du kör setup.exe /q RUNAUTOUPDATESILENTLY=TRUE från kommandoraden. (399265)

  • Problem

    När du kör kommandot setup.exe CMASOURCEDIR="<drive>:\Documents and Settings\<username>\Desktop\" från kommandoraden för att installera produkten och importera Sitelist.xml-filen från skrivbordet, slutförs installationen utan problem men det går inte att importera Sitelist.xml-filen.

    Lösning: Använd en Sitelist.xml-fil som skapats av ePolicy Orchestrator Agent 4.0. Om McAfee Agent 4.0-installationen uppgraderade en tidigare installation av ePolicy Orchestrator Agent 3.6.1 skapas en korrekt Sitelist.xml-fil som kan importeras. (419467)

      1. Klicka på Menu | Policy | Policy Catalog (Meny | Princip | Principkatalog). I listan med produkter väljer du sedan VirusScan Enterprise 8.8.0. I kategorilistan visas principkategorierna för VirusScan Enterprise 8.8.0.

      4. På sidan för allmänna åtkomstprinciper klickar du på fliken ScriptScan.
      5. Avmarkera kryssrutan Enable scanning of scripts (Aktivera sökning av skript).
      6. Spara konfigurationen genom att klicka på Save (Spara).
    • ePolicy Orchestrator 4.0
      1. Klicka på Systems | Policy Catalog (System | Principkatalog). I listan med produkter väljer du sedan VirusScan Enterprise 8.8.0. I kategorilistan visas principkategorierna för VirusScan Enterprise 8.8.0.
      2. Välj On-Access General Policies (Allmänna principer för åtkomst) i listan Category (Kategori) och klicka på Edit Setting (Redigera inställning) i åtgärdskolumnen så att principkonfigurationssidan öppnas.
      3. I listan Settings for (Inställningar för) väljer du Workstation (Arbetsstation) eller Server (Server).
      4. På sidan för allmänna åtkomstprinciper klickar du på fliken ScriptScan.
      5. Avmarkera kryssrutan Enable scanning of scripts (Aktivera sökning av skript).
      6. Spara konfigurationen genom att klicka på Save (Spara).
    • VirusScan-konsolen
      1. I aktivitetslistan högerklickar du på Sökning vid användning och klickar sedan på Egenskaper för att öppna dialogrutan.
      2. På sidan Egenskaper för sökning vid användning klickar du på Allmänna inställningar till vänster, och klickar på fliken ScriptScan.
      3. Avmarkera kryssrutan Aktivera sökning av skript.
      4. Spara konfigurationen genom att klicka på OK.
