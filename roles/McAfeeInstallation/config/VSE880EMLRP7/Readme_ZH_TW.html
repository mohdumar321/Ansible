關於本文件

感謝您使用 VirusScan Enterprise 軟體。本文件包含有關此版本的重要資訊。McAfee 強烈建議您詳細閱讀本文件。

注意: McAfee 不支援對軟體的預發佈版本進行自動升級。若要升級為軟體的生產版本，必須先解除安裝軟體的現有版本。

VirusScan Enterprise 8.8 版本使用下列內容：

  • 引擎版本： 5400.1158
  • 偵測定義 (DAT)：安裝此版本的 VirusScan Enterprise 軟體後，該軟體會自動下載最新版本的 DAT 檔案。之前，DAT 檔案包含在安裝可執行檔中。

新功能

VirusScan Enterprise 8.8 軟體版本中的新功能和更新的功能。

效能提高

此版本減少了：
  • 常駐掃描時間
  • 按指定掃描時間
  • 系統啟動時間

此版本會將之前掃描的檔案快取到公用位置，以減少重複檔案掃描，並提高效能。

Windows Office 2010 支援

此版本支援所有 Windows Office 2010 應用程式，特別是新增了 Microsoft Outlook 2010 電子郵件和附件檔案的常駐掃描。

ScriptScan 排除項目支援

可輕鬆新增 ScriptScan 排除項目 - 來自已知安全網站之受信任指令碼的 URL，以減少掃描，並提高效能。ePolicy Orchestrator 4.0 或更新版本以及 VirusScan 主控台中提供此功能。之前，這些排除的 URL 需要手動執行登錄編輯。

遷移 ePolicy Orchestrator 中的原則與事件

  • 問題

    ePolicy Orchestrator 遷移工具可能無法從使用 Windows NT 驗證存取 ePolicy Orchestrator 的ePolicy Orchestrator 伺服器遷移現有原則和工作。有時，ePolicy Orchestrator 遷移工具需要把 SQL 管理員密碼作為 ePolicy Orchestrator 遷移工具的參數傳遞。

    解決方法：從命令列中執行 ePOPolicyMigration.exe 命令，並新增 /PASSWORD 參數和 SQL 管理員密碼。例如，鍵入以下內容：[路徑]\epopolicymigration.exe /password=SQL 管理員密碼

    如需有關在命令列中使用 ePolicy Orchestrator /PASSWORD 參數的資訊，請參閱《McAfee VirusScan Enterprise 8.8.0 安裝指南》。 (611047)

與其他產品的相容性

  • 問題
    ePolicy Orchestrator 使用者介面的 [優先執行此動作] 清單中少了 [提示動作] 工作選項。例如，如果您要使用 ePolicy Orchestrator 使用者介面設定一個按指定掃描工作，並且：
    • 按一下 [系統樹狀目錄 | 用戶端工作 | VSE 8.8.0 按指定掃描工作]
    • 按一下 [編輯設定]、[組態] 和 [動作] 標籤

    此對話方塊的兩個 [優先執行此動作] 清單中都沒有 [提示動作] 工作選項。

    但是在 VirusScan 主控台使用者介面中有 [提示動作] 工作選項。例如：
    • 如果您連按兩下某個按指定掃描工作，請於 [按指定掃描內容] 對話方塊中按一下 [動作] 標籤。
    • 即可從 [優先執行此動作] 清單中選取 [提示動作] 工作。

    解決方法：無。這是由 ePolicy Orchestrator 管理的掃描工作，所以執行按指定掃描工作時不會向使用者提供使用者介面。因此，在提示使用者執行動作方面，不提供任何組態。 (590803)

  • 問題

    從 VirusScan Enterprise 8.7 成功升級到 VirusScan Enterprise 8.8 之後，ePolicy Orchestrator 4.5 [系統詳細資料] 對話方塊的 [已安裝產品] 列中，沒有列出 VirusScan Enterprise 8.8。 (620509)

  • 問題
    在安裝了 McAfee Network Access Control 3.0 的系統上安裝 VirusScan Enterprise 8.8 後，可能會發生故障，導致無法識別 McAfee 服務。您可能會看到以下徵兆：
    • 如果您在重新啟動之前執行更新工作，則會記錄「指定的驅動程式無效」的訊息。
    • 針對一個或兩個產品記錄「驅動程式載入失敗」的訊息。
    • 本機系統應用程式事件記錄包含事件 ID 5004。

    解決方法：解除安裝有問題的產品，重新啟動系統，然後重新安裝產品。

  • 問題

    NVIDIA ® 驅動程式可能會引發效能問題或系統故障。VirusScan Enterprise 8.8 可能會以 100% CPU 使用率執行或導致系統發生故障。如需有關此問題的詳細資訊，請參閱知識庫 (KnowledgeBase) 文章 KB51688 與 KB52367。 (388060)

  • 問題

    執行 VirusScan Enterprise 8.8 安裝程式無法完全移除 CA Anti-Virus 7.0 版和 7.1 版的全部程式目錄結構和登錄機碼項目。

    解決方法：無。這是 CA Anti-Virus 解除安裝程式的問題。 (625703)

  • 問題

    在獨立用戶端系統上，如果也安裝了 Panda Anti-Virus 2008，VirusScan Enterprise 8.8 將無法完成初始產品更新。

    執行 VSESetup.exe 時，如果選取選項 [否]，不移除 Panda Anti-Virus 2008 軟體，則在最後階段VirusScan Enterprise 8.8 執行產品更新時，更新對話方塊可能會停滯 20 分鐘或更久。

    解決方法：執行 VSESetup.exe 時，選取選項 [是]，移除 Panda Anti-Virus 2008 軟體。 (626009)

  • 問題

    在由 ePolicy Orchestrator 管理的用戶端上，如果執行 VSESetup.exe 並選取選項移除偵測到的防毒軟體，某些 Symantec 防毒檔案、目錄和登錄機碼仍會保留在系統上。

    解決方法：無。 (629265)

更新

  • 問題
    更新記錄可能會報告「更新失敗」，但不指示任何其他錯誤。修補程式、DAT 和引擎的更新功能需要足夠的可用空間和記憶體來擴充升級套件。您必須符合以下最小空間需求，才能成功更新：
    • 125 MB 的可用硬碟空間
    • 250 MB 的總可用硬碟和記憶體空間
      附註: 也可以使用 Windows 虛擬記憶體建立額外的記憶體空間，但該總可用空間必須達到 250 MB。

    此硬碟和記憶體需求對於經常以最小系統需求執行的內嵌和虛擬系統非常重要。 (625749)

  • 問題

    未將存放庫存取失敗記錄在 VirusScan Enterprise 8.8 更新記錄中，但記錄在 McAfee Agent 記錄中。McAfee Agent 記錄的預設位置為：<磁碟機>:\Documents and Settings\All Users\Application Data\McAfee\Common Framework\mcscript.log。 (384749)

  • 問題

    從命令列執行的無訊息更新工作仍會顯示進度對話方塊。安裝已成功完成，但當您從命令列執行 setup.exe /q RUNAUTOUPDATESILENTLY=TRUE 時仍顯示 [更新進行中] 對話方塊。 (399265)

  • 問題

    當您從命令列中執行 setup.exe CMASOURCEDIR="<磁碟機>:\Documents and Settings\<使用者名稱>\Desktop\" 命令來安裝產品，並從桌面匯入 Sitelist.xml 檔案時，安裝作業會成功完成，但無法匯入 Sitelist.xml 檔案。

    解決方法：使用由 ePolicy Orchestrator Agent 4.0 建立的 Sitelist.xml 檔案。如果 McAfee Agent 4.0 安裝升級了先前安裝的 ePolicy Orchestrator Agent 3.6.1，則會產生可供匯入的正確的 Sitelist.xml 檔案。 (419467)

遠端主控台

  • 問題

    使用遠端主控台功能連線至 Windows Vista 或更新版本的系統時，您無法為遠端系統開啟 [存取保護]、[緩衝區溢位] 和 [Quarantine Manager] 對話方塊，並且不會顯示任何錯誤。這是因為 Windows Vista 和更新版本的系統不會自動啟動遠端主控台功能所需的遠端登錄服務。

    解決方法：啟動遠端主控台功能之前，使用下列方法在遠端系統上手動啟動遠端登錄服務：
    • 按一下 [開始 | 執行]，然後鍵入 services.msc。系統會顯示 [服務] 對話方塊。
    • 在 [服務 (本機) | 名稱] 欄中的 [遠端登錄] 上按一下滑鼠右鍵，然後按一下 [啟動]。
    (581439)
  • 問題

    使用遠端主控台功能在安裝了 Windows Server 2008 的系統上開啟存取保護內容時，可能需要花費數分鐘的時間來開啟連線。 (392825)

    2. 當 Windows UAC 視窗顯示 [您是否要允許下列程式變更這部電腦？] 提示時，即使您按一下 [否] 來捨棄變更，系統仍會發生變更。

    解決方法：無。這是 Windows UAC 的問題，沒有解決方法。 (627058)

  • 問題

    停用常駐掃描不會停用 ScriptScan。對於 VirusScan Enterprise 8.7 和之前的版本，如果您停用常駐掃描，也會停用 ScriptScan 功能。對於 VirusScan Enterprise 8.8，如果您停用常駐掃描，ScriptScan 會繼續執行。

    解決方法：使用下列其中一個使用者介面停用 VirusScan Enterprise 8.8 ScriptScan 功能：
    • ePolicy Orchestrator 4.5 或 4.6
      1. 按一下 [功能表 | 原則 | 原則目錄]，然後從 [產品] 清單中選取 [VirusScan Enterprise 8.8.0]。[類別] 清單會顯示 VirusScan Enterprise 8.8.0 的原則類別。
      2. 從 [類別] 清單中，選取 [常駐一般原則]，並從 [動作] 欄中，按一下 [編輯設定] 以開啟 [原則組態] 頁面。
      3. 從 [設定適用於] 清單中，選取 [工作站] 或 [伺服器]。
      4. 從 [常駐一般原則] 頁面中，按一下 [ScriptScan] 標籤。
      5. 取消選取 [啟用指令碼掃描] 核取方塊。
      6. 按一下 [儲存] 以擷取組態變更。
    • ePolicy Orchestrator 4.0
      1. 按一下 [系統 | 原則目錄]，然後從 [產品] 清單中選取 [VirusScan Enterprise 8.8.0]。[類別] 清單會顯示 VirusScan Enterprise 8.8.0 的原則類別。
      2. 從 [類別] 清單中，選取 [常駐一般原則]，並從 [動作] 欄中，按一下 [編輯設定] 以開啟 [原則組態] 頁面。
      3. 從 [設定適用於] 清單中，選取 [工作站] 或 [伺服器]。
      4. 從 [常駐一般原則] 頁面中，按一下 [ScriptScan] 標籤。
      5. 取消選取 [啟用指令碼掃描] 核取方塊。
      6. 按一下 [儲存] 以擷取組態變更。
    • VirusScan 主控台
      1. 從 [工作] 清單中，在 [常駐掃描程式] 上按一下滑鼠右鍵，然後按一下 [內容] 以開啟對話方塊。
      2. 從 [常駐掃描內容] 頁面中，按一下左窗格中的 [一般設定]，然後按一下 [ScriptScan] 標籤。
      3. 取消選取 [啟用指令碼掃描] 核取方塊。
      4. 按一下 [確定] 以擷取組態變更。

說明文件

  • 問題

    發佈《McAfee VirusScan Enterprise 8.8 軟體產品指南》和說明後，〈設定一般設定〉一節中的資訊發生變更。

    在每個使用者介面主控台中，已修改下列步驟以包含附註：

    在 [ScriptScan] 標籤上，啟用 ScriptScan，並設定要從掃描排除的任何處理程序或 URL。
    附註: 停用常駐掃描不會停用 ScriptScan。對於 VirusScan Enterprise 8.7 與之前的版本，停用常駐掃描會停用 ScriptScan 功能。對於 VirusScan Enterprise 8.8，停用常駐掃描 ScriptScan 不會停用 ScriptScan。若要停用 ScriptScan，請取消選取 [啟用指令碼掃描] 核取方塊。
  • 問題

    發佈《McAfee VirusScan Enterprise 8.8 軟體產品指南》和說明後，[常駐掃描一般原則說明] 標籤中的資訊發生了變更。

    [掃描] 選項中已加入以下資訊：

    受信任的安裝程式 — 指定是否對 MSI 檔案 (由 msiexec.exe 安裝，由 McAfee 或 Microsoft 簽署) 或 Windows 受信任的安裝程式服務檔案進行常駐掃描：
    • 不選取，不掃描由 MSI 或 Windows 受信任的安裝程式服務安裝的檔案。預設 = 停用
      附註: 不選取該選項可提高大型 Microsoft 應用程式安裝程式的效能。
    • 選取，掃描由 MSI 或 Windows 受信任的安裝程式服務安裝的檔案。
      附註: 掃描內容不包括命令列安裝所包含的或列入白名單的合併或範本 MSI 檔案。
